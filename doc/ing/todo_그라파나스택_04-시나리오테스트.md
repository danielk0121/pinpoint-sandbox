# 4단계: 시나리오 테스트

## 목표

구축된 스택에서 실제 모니터링 시나리오를 실행하고,
트레이스/메트릭/로그 연계가 정상 동작하는지 검증한다.

---

## 사전 준비

- [ ] 1~3단계 완료 확인 (로컬 환경 구축, Java/Go 앱 연동)

---

## 작업 목록

### 시나리오 1: 느린 트랜잭션 추적

- [ ] `/slow` 엔드포인트 반복 호출
- [ ] Jaeger UI에서 응답시간 기준 트레이스 필터링
- [ ] 느린 스팬 구간 특정
- [ ] Loki에서 해당 TraceID로 로그 연계 조회

### 시나리오 2: 에러 트랜잭션 추적

- [ ] `/error` 엔드포인트 호출
- [ ] Jaeger UI에서 에러 트레이스 확인
- [ ] Loki에서 에러 로그 연계 조회

### 시나리오 3: Grafana 대시보드 구성

- [ ] Java 앱 대시보드
  - JVM 메모리 사용량
  - GC 횟수/시간
  - HTTP 요청 수 / 응답시간 / 에러율
- [ ] Go 앱 대시보드
  - HTTP 요청 수 / 응답시간 / 에러율
  - Goroutine 수
- [ ] 대시보드 JSON 파일로 export 및 저장

### 시나리오 4: 알림 설정

- [ ] Grafana Alert Rule 생성
  - 응답시간 1초 초과 시 알림
  - 에러율 5% 초과 시 알림
- [ ] 알림 채널 설정 (이메일 또는 Slack)
- [ ] 임계치 초과 상황 재현 후 알림 수신 확인

### 최종 검증

- [ ] Java 앱 트레이스/메트릭/로그 통합 조회 정상 동작
- [ ] Go 앱 트레이스/메트릭/로그 통합 조회 정상 동작
- [ ] TraceID 기반 트레이스 ↔ 로그 연계 정상 동작
- [ ] 대시보드 및 알림 정상 동작
