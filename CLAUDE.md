# Pinpoint Sandbox

Naver Pinpoint APM 학습 및 테스트 프로젝트.

## 작업 관리

`doc/` 폴더를 애자일 방식으로 활용:
- `doc/ing/` — 진행 중인 작업 (todo_xxx.md)
- `doc/hold/` — 보류 중인 작업 (todo_xxx.md)
- `doc/done/` — 완료된 작업 (todo_xxx.md)

파일명 규칙: `todo_<작업명>.md`

## 참고 자료

`ref/` 폴더에 리서치 결과물 보관:
- 스크린샷 캡처 이미지
- 리서치 결과 문서

## 기본 규칙
- `~/` 유저 홈 설정을 따르지 않는다.
- 모든 문서는 한국어로 작성한다. 최대한 한국어를 사용한다.
- 코딩에서는 주로 영어를 사용하되, 주석이나 문자열은 한국어를 사용한다.
- 문서에서 가급적 표(table) 형태 대신 목록(list) 형태를 사용한다.
- 커밋은 `git commit -m "메시지"` 형태로 간략하게 진행한다. cat 또는 HEREDOC을 사용하지 않는다.
- 문서 파일명은 한글을 사용한다. (예: `todo_아키텍처이해.md`, `research_핀포인트개요.md`)
- 문서 파일 작업이 끝나면 항상 커밋 후 푸시한다.
